# Woodoku Unityベース開発TODOリスト

AIの環境として機能させることを前提とした、UnityでのWoodokuベース開発の具体的なTODOリストです。内部のデータ管理と画面の描画を明確に分けることを意識して進めます。

## 1. プロジェクトとデータ構造の準備
- [x] Unityで新規2Dプロジェクトを作成する。
- [x] `BoardData` クラスを作成し、9x9の盤面状態を管理する二次元配列（`int[,] board = new int[9, 9];`）を定義する。
- [x] `BlockData` を `ScriptableObject` として作成し、ブロックの形状を `Vector2Int` の配列（またはリスト）で定義できるようにする。
- [x] 代表的なブロック（1x1、3x3、L字、I字など）の `ScriptableObject` アセットを数種類作成する。

## 2. 盤面（グリッド）の描画設定
- [x] 1マス分を表す正方形のSpriteを用意し、`Cell` プレハブを作成する。
- [x] `Cell` プレハブに、現在の状態（空き、ブロックあり）に応じて色を変更するスクリプトをアタッチする。
- [x] `BoardUI` クラスを作成し、ゲーム開始時に `Cell` プレハブを9x9のグリッド状に生成・配置する処理を記述する（`GridLayoutGroup` やスクリプトでの座標計算を利用）。

## 3. 手札ブロックの生成と表示
- [ ] **手札ブロックのピース（`BlockPiece`）プレハブ作成**
  - 盤面の `Cell` とは別に、手札ブロック用の1マス分を表すUIプレハブを作成する（画像自体は同じでOK）。
- [ ] **手札エリア（スロット）のUI構築**
  - 画面下部に `HandArea`（`HorizontalLayoutGroup`などを適用）を作成し、ブロックを配置するための見えない枠（スロット）を3つ均等に配置する。
- [ ] **1つのブロック形状を構築する `HandBlock` クラスの作成**
  - `HandBlock` プレハブを作成し、このスクリプトをアタッチする。
  - `BlockData`（ScriptableObject）を受け取り、その `Vector2Int` の座標データに基づいて、自身の子オブジェクトとして `BlockPiece` を必要な数だけ生成・配置する処理を作る。
  - スロット（枠）の中央に綺麗に収まるように、ブロック全体のピボット（中心点）やスケールを調整する処理を入れる。
- [ ] **3つの手札全体を管理する `HandManager` クラスの作成**
  - 登録されている複数の `BlockData` アセットの中から、ランダムに3つを選出する。
  - 3つの手札スロットにそれぞれ `HandBlock` プレハブを生成し、選出した `BlockData` を渡して画面に表示させる。

## 4. ドラッグ＆ドロップ機能の実装
- [ ] 手札ブロックの親オブジェクトに `EventTrigger` または `IDragHandler`, `IBeginDragHandler`, `IEndDragHandler` を実装したスクリプトをアタッチする。
- [ ] ドラッグ開始時（`OnBeginDrag`）にブロックが指やマウスポインタに追従するようにする。
- [ ] ドラッグ中（`OnDrag`）はブロックの描画レイヤーを最前面にし、指に隠れないよう少し上方にオフセットをつける。
- [ ] ドロップ時（`OnEndDrag`）に、現在の画面座標から盤面（9x9）のどのマスのインデックスに近いかを計算する処理を作る。

## 5. 配置判定とデータ更新
- [ ] ドロップされたインデックスを基準に、ブロックの形状（`Vector2Int` の配列）を盤面の座標に変換する。
- [ ] 変換した座標が「盤面（0〜8）の範囲内に収まっているか」を判定する（配列の範囲外アクセス対策）。
- [ ] 変換した座標の盤面データ（`board` 配列）がすべて `0`（空き）であるかを判定する。
- [ ] 配置可能であれば、該当する盤面データを `1` に更新し、画面の `Cell` プレハブの色を更新する。
- [ ] 配置不可能（範囲外、または重なりがある）であれば、ブロックを元の手札エリアの位置に戻す（Tweenアニメーションなどを使うと自然）。

## 6. 消去判定（ラインと3x3ブロック）の実装
- [ ] ブロック配置直後に、盤面全体をスキャンする処理を `BoardData` クラスに実装する。
- [ ] **行の判定:** 各行（Y=0〜8）について、X=0〜8がすべて `1` であるかを確認し、消去対象リストに追加する。
- [ ] **列の判定:** 各列（X=0〜8）について、Y=0〜8がすべて `1` であるかを確認し、消去対象リストに追加する。
- [ ] **3x3エリアの判定:** 盤面を9つの3x3ブロック（左上、上中央など）に分け、それぞれの中がすべて `1` であるかを確認し、消去対象リストに追加する。
- [ ] 消去対象リストに含まれる座標の盤面データを `0` に戻し、画面の `Cell` プレハブを空き状態の色に戻す。

## 7. ゲームサイクルとゲームオーバー判定
- [ ] 手札のブロックが配置されて消費されたら、該当のブロックUIを削除（または非表示）にする。
- [ ] 3つのブロックがすべて消費されたら、`HandManager` を呼び出して新たに3つのブロックを生成する。
- [ ] ブロックが新たに補充された時、およびブロックを盤面に配置した直後に、**手詰まり判定**を行う。
  - 現在手札に残っている各ブロックについて、盤面（0,0 から 8,8）のすべての座標に配置可能か全探索でシミュレーションする。
  - 1箇所でも置ける場所があればゲーム続行。
  - どのブロックも、どこにも置けなくなったら「ゲームオーバー」の処理（UI表示など）を呼び出す。
